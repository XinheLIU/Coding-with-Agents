
这两年，AI Coder 渐渐侵蚀了属于我们码农的场域，我们把自己升级叫开发者（或架构师）——开发产品的人（谁来编码并不重要，关键是程序开发由谁来掌控，这个词会让我们暂时感觉更安全）。但在 Claude Coding 时代，我更愿意用一个老词：极客（Geek）。
为什么？因为 Claude Code 这个级别的 AI 正在让编程能力民主化。以前，想把一个创意变成产品，需要具备这些条件。
会写代码（技术门槛）
有足够的时间（执行成本）
能处理各种工程细节（经验门槛）
现在，这三个门槛都在急剧降低。不会写代码？AI 可以帮你写。没有时间？AI 可以 24 小时工作。不懂工程细节？AI 知道最佳实践。
这意味着什么？能力的天平正在从“执行力”向“创意力”和“组织力”倾斜。—— 而这些能力往往并不属于传统的“程序员”，而是程序员或开发者的加强版——也就是极客。
以前，有想法但不会写代码的人，需要找程序员合作。现在，他可以直接和 AI 合作。以前，产品经理和工程师之间有一道翻译的鸿沟。现在，AI 可以做那个翻译。以前，一个人做不了什么大项目。现在，一个人 + AI 团队，可以挑战以前需要一个团队才能完成的事情。

PM 负责“想”，工程师负责“做”。沟通成本极高。但现在，如果一个 PM 懂业务逻辑，他完全可以直接用 Claude Code 把想法变成现实，而不需要再写长达几十页的 PRD。反之，如果一个工程师有了好点子，他也不需要等 PM 来排期，自己就能独立完成闭环。“软件工程师”这个头衔将变得狭隘，取而代之的是“建设者”（Builder）。

Boris 预测，到今年年底，每个人都会成为产品经理，同时每个人也都会写代码。传统的职能划分正在崩塌。

以前，PM 负责“想”，工程师负责“做”。沟通成本极高。但现在，如果一个 PM 懂业务逻辑，他完全可以直接用 Claude Code 把想法变成现实，而不需要再写长达几十页的 PRD。反之，如果一个工程师有了好点子，他也不需要等 PM 来排期，自己就能独立完成闭环。“软件工程师”这个头衔将变得狭隘，取而代之的是“建设者”（Builder）。

“我认为到今年年底，每个人都会成为产品经理，每个人都会编写代码。‘软件工程师’这个头衔将开始消失，它将被‘建设者’所取代，这对很多人来说将是痛苦的。”

无论你现在的职位是什么，立刻开始培养你的“端到端”交付能力。PM 要学会用 AI 独立跑通原型，工程师要学会主动挖掘用户需求。未来的组织结构将趋向于“超级个体化”，只懂一端的人将被淘汰。

为了应对这种剧烈的变化，Boris 在面试和带领团队时，确立了三项原则与新成员分享。其核心指向了适应性、系统思考和主人翁意识。

在编程被解决的时代，团队不再需要“螺丝钉”，而是需要能在不确定性中自我导航的“建设者”。这些原则要求团队成员停止把自己定位为单纯的“技术实现者”，而是把自己当成业务的负责人。他们需要具备极强的业务敏感度，能够迅速理解用户痛点，并利用 AI 杠杆以极低的成本解决问题。
