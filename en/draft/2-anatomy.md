
Claude Code 出现时，包括我在内的很多人都明确意识到：真正被重塑的，已经不是“写代码”，而是 Vibe Coding 这件事本身。
在 Claude Code 中，你更容易这样描述任务：
“这个模块现在可读性差、测试覆盖不足，还可能有安全隐患。
我希望你帮我系统性地重构它，但不要影响对外接口。”
你会发现，Claude Code 非常懂工程（并不仅仅是 Claude 模型本身代码能力强）。他会自动规划任务，自动拆解任务，自动雇佣一堆 Sub Agents 做事情，自动确认什么时候要引入人类决策。你不是在一句一句“教它写代码”，而是在和一个工程系统对齐节奏。

现在我们使用 Claude Code 等 AI Coding 工具编码，其实你不再只是把自然语言翻译成代码， 而是在做设计，具体来说是做三件更高级的事情：
拆解问题
分配任务
组织多个智能体协作完成目标
这已经不是“编程工具”的范畴了，而是一种新的工作范式


而 Claude Code 还支持另一种模式：
用户 → 配置 Agent → Agent 自主工作 → 自动完成任务
这是主动驾驭——你设计，它执行。就像我们编写程序，程序自动运行。

| 场景     | 被动使用                     | 主动驾驭                                               |
| -------- | ---------------------------- | ------------------------------------------------------ |
| 代码审查 | 帮我看看这段代码有没有问题   | 配置一个 code-reviewer 子代理，每次 PR 自动审查        |
| 跑测试   | 帮我跑一下测试               | 配置 Hook，每次修改代码自动跑测试                      |
| 写文档   | 帮我写个 README              | 配置 Skill，Claude 识别文档需求时自动应用文档规范      |

# 理解 Claude Code：从“使用者”到“驾驭者”

Claude Code 的学习路径类似于学车。你可以只是“会开车”，也可以深入掌握并改造整辆车：

| 角色     | 能力范围                                       |
| -------- | ---------------------------------------------- |
| 使用者   | 知道如何让 Claude 帮写代码，学会基本提问方式   |
| 驾驭者   | 理解记忆系统、子代理、技能包、钩子的原理，能搭建和自定义智能工作流 |

---

## Claude Code 技术分层结构

Claude Code 的能力分为四个技术层级：

| 层级         | 关键概念/组件             | 说明与典型应用                       |
| ------------ | ------------------------- | ------------------------------------ |
| 基础层       | Memory（记忆系统）        | 以 CLAUDE.md 作为长期团队知识库：存放代码风格规范、架构准则、项目禁区等 |
| 扩展层       | Commands、Skills、SubAgents、Hooks | 四大能力组件，承载具体智能体功能       |
| 集成层       | Headless、MCP             | 连接外部世界，跨平台自动化与工具集成   |
| 编程接口层   | Agent SDK                 | 深度自定义，代码级扩展和二次开发支持   |

---

### 各层能力详解

#### 1. 基础层：Memory（记忆系统）

- 本质是一个团队级的持久“手册”，例如 CLAUDE.md。
- 存储风格规范、Git 提交流程、架构说明、禁区提示等。

#### 2. 扩展层：四大核心能力组件

| 组件         | 触发方式         | 本质定位                        | 谁触发         | 决策权在谁 | 适用场景                   | 适合解决的问题                           | 不适合做什么                   | 说明／典型一句话总结                                  |
| ------------ | ---------------- | ------------------------------- | -------------- | ---------- | -------------------------- | ---------------------------------------- | ------------------------------ | -------------------------------------------------- |
| Commands（斜杠命令）   | /斜杠命令手动      | 人工触发的标准操作流程（SOP）        | 用户 /command  | 人         | 标准流程、统一操作              | 固定流程、可复现操作、团队统一规范               | 依赖语义判断、需要灵活决策的任务         | “按我写好的步骤，严格执行”<br>适合需复用/审计的固定流程        |
| Skills（技能）         | 语义自动判断        | 模型内部的专家行为模式                | Claude 自动    | 模型        | 需专家型判断任务                  | 领域能力、上下文判断、策略可变的问题               | 纯机械、无判断空间的操作             | “我判断这是我的专业领域，让我像专家一样来”<br>内部封装行为逻辑和触发策略 |
| SubAgents（子代理）    | 显式或自动决策      | 独立上下文与权限的执行单元              | Claude 或用户  | 架构设计     | 隔离、批量子任务                  | 高噪声任务、权限隔离、并行处理                   | 简单一步就能完成的任务               | “这件事我不亲自干，交给专人处理”<br>用于高噪声/高权限/定向任务    |
| Hooks（钩子）          | 事件自动触发        | 事件触发的强制规则                      | 系统事件        | 系统事件    | 自动化检查/守卫任务                | 安全、合规、自动化校验                           | 需要权衡、讨论、模糊判断的事情         | “不管你想不想，这条规则必须执行”<br>如格式化、安全监测、日志        |

**Commands vs Skills：**
- Commands 着重于确定性和复用性，比如统一提交信息、部署指令。
- Skills 应对需上下文理解、领域判断的复杂任务，例如自动代码安全检查。

**SubAgents：**
- 独立子代理可批量并行，如单独跑测试、日志检索，仅回报主 Agent。

**Hooks：**
- 自动化防线，如编辑前自动安全检查，发现问题即拦截流程。

#### 3. 集成层：连接外部世界

| 组件          | 功能示例                | 用法简介                           |
| ------------- | ---------------------- | ---------------------------------- |
| Headless 无头 | CI/CD 全自动修修补补   | `claude --headless ...` 脚本集成   |
| MCP           | 外部系统互通           | 支持 API、数据库、Jira 等对接      |

#### 4. 编程接口层：Agent SDK

- 深度自定义扩展能力。例如用 Python SDK 全流程驱动 Claude：
  ```python
  from claude_sdk import ClaudeSDKClient
  client = ClaudeSDKClient()
  result = client.query(
      prompt="Review this code for security issues",
      tools=["Read", "Grep"]
  )
  ```

## 选用建议

**是否需要 Claude 具备某种能力？**
    *   **是**：
        *   若希望**用户手动触发** → 使用 **Commands**
        *   若希望 **Claude 自动识别并应用** → 使用 **Skills**
    *   **否**：根据具体场景选择以下方案：
        *   需要隔离执行/减少上下文 → **SubAgents**
        *   需要在工具执行时自动检查 → **Hooks**
        *   需要连接外部系统 → **MCP**
        *   需要 CI/CD 集成 → **Headless**
        *   需要完全编程控制 → **Agent SDK**


| 需求场景 | 推荐方案 | 核心特点 |
| :--- | :--- | :--- |
| 手动触发特定任务 | Commands | 用户主动调用，执行预设指令 |
| 自动识别并应用能力 | Skills | Claude 自主判断，按需启用 |
| 隔离执行、减少上下文 | SubAgents | 子代理独立执行，降低主代理负担 |
| 工具执行时自动检查 | Hooks | 嵌入工具执行流，实现自动化校验 |
| 连接外部系统 | MCP | 打通 Claude 与外部服务/数据 |
| CI/CD 流程集成 | Headless | 无界面模式，适配自动化流水线 |
| 完全编程控制 | Agent SDK | 深度定制，实现复杂智能体逻辑 |
