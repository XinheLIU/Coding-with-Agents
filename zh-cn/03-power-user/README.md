# 第三章：成为高级用户

> *使用编程智能体与精通它之间的差异，就像会开车与理解引擎、道路和天气之间的差异。*

[第一章](../01-prompt/README.md)和[第二章](../02-anatomy/README.md)为你奠定了基础：如何向智能体发出指令，以及它们的工作原理。本章将带你从"我会用这个工具"迈向"我能编排它"。这里的技巧代表了一个从 AI 获得 2 倍效能的开发者与获得 10 倍效能的开发者之间的差距。

这是本书中篇幅最大的章节——这是有意为之的。高级用户技巧是复利回报所在，每一节都建立在上一节的基础之上。以下是其结构：

1. **平台深度解析** — [Claude Code 作为 AI 智能体框架](claude-code.md)是本章的开篇。它阐释了*为什么*我们以 Claude Code 作为主要视角——其四层架构（内存、扩展、集成、SDK）直接映射至第二章的三大支柱，掌握它能为你理解任何智能体系统建立可迁移的直觉。本节还涵盖了使架构具体化的实现细节（微提示词、命令安全、自动压缩）。

2. **委托模式** — [子智能体](sub-agents.md)介绍了为隔离任务生成具有特定范围的临时智能体的概念——如何避免上下文污染并并行化工作。[智能体团队](agent-teams.md)将此延伸至协调多个智能体处理更大型项目。

3. **扩展智能体** — [技能](skills.md)和 [Hooks](hooks.md)展示如何为智能体赋予新能力，并通过事件驱动的自动化将其接入开发工作流。[为智能体构建工具](building-tools.md)涵盖 MCP 服务器、自定义工具和环境配置。

4. **综合实践** — [系统性思维](systematic-thinking.md)将一切融会贯通。它教授将技能、子智能体、Hooks 等组合成生产级智能体系统的*组合原则*——包括治理设计、上下文隔离以及完整的架构示例。

---

**你已走过的路：** [第二章](../02-anatomy/README.md)为你建立了智能体工作方式的心智模型。现在你拥有了利用这种理解的具体技巧。

**下一步：** 个人精通固然强大，但真正的影响力来自于在团队中推广这些实践。[第四章](../04-team/README.md)直面团队协作中与智能体相关的难题：共享上下文、以 AI 速度进行代码审查，以及在输出量激增时保持高质量的测试策略。
