# 第二章：编程智能体解剖

> *你无法有效驾驭一个自己不了解的工具。本章将帮助你建立正确的心智模型。*

在[第一章](../01-prompt/README.md)中，你学习了如何向编程智能体下提示词。但要做好这件事，光靠一些套路是不够的——你还需要理解*为什么*某些方法有效，而另一些会失败。本章通过剖析编程智能体的本质，来打牢这一基础：它们究竟是什么，如何做出决策，以及在哪些地方容易出问题。

本章围绕三个层次的理解展开：

1. **这台机器如何运转** — [编程智能体工作原理](how-agents-work.md) 追溯了从内联补全到智能体化编程的演进历程，并解释了每个智能体共有的三大支柱：系统提示词、工具以及上下文策略。理解这三大支柱，是在事情出错时调试智能体行为的关键所在——也为[第三章](../03-power-user/claude-code.md)深入剖析某一具体智能体（Claude Code）如何将这些支柱组织成完整架构铺垫了基础。

2. **该给它们多大自由度** — [自治级别与人工干预](autonomy-levels.md) 提供了一个框架，帮你判断何时可以让智能体自由运转，何时需要紧紧把控。核心原则是：低风险的重复性工作给予高度自治；业务关键逻辑则需严格监督。本节还整理了最常见的失败模式及其预防方法。

3. **两种构建哲学** — [Vibe Coding 与 Spec Coding](vibe-vs-spec.md) 描绘了 AI 时代开发方式的完整光谱——从快速、直觉驱动的"vibe coding"，到严谨的、规格先行的工程化开发。这两者并非对立的意识形态，而是同一门技艺的不同阶段。知道何时该用哪种方式，正是高效开发者与陷入困境者之间的分水岭。

---

**接下来：** 现在你已经理解了这台机器，[第三章](../03-power-user/README.md) 将带你把它推向极限。你将学习上下文管理、子智能体、hooks、技能以及工作流设计模式，从一个称职的用户蜕变为真正的高级用户。
