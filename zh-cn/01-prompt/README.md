# 第一章：如何提示编程智能体

> *在 AI 编程时代，单一最高杠杆的技能不是写代码——而是写出能让智能体编写正确代码的提示词。*

旅程从这里开始。在理解智能体的工作原理、优化其上下文或在团队中大规模使用之前，你需要掌握一项基础技能：如何向 AI 编程智能体清晰地传达意图。本章是你的入门跑道。

我们围绕三个递进式问题来组织本章内容：

1. **发生了什么变化？** — AI 编程智能体不只是更快的自动补全。它们从根本上改变了你作为开发者*如何分配时间*。[编程智能体如何改变开发方式](how-agents-change-dev.md)重新定义了开发者角色——从编写代码转变为指挥智能体，并展示了新的内循环是什么样的。

2. **如何与它们沟通？** — 了解全貌还不够，你需要实用技巧。[提示原则](prompting-principles.md)提炼了写提示词、CLAUDE.md 文件和系统指令时影响最大的模式——先给上下文再给指令、明确说明约束条件，以及何时应该重新开始对话。

3. **如何让效果持久？** — 如果每次会话都要重复，再好的提示词也是浪费。[AGENTS.md](agents-md.md) 介绍了持久化智能体记忆的概念——一个单独的文件，能将"Claude，我们用 pnpm"从每次会话的提醒，变成团队的组织知识。
